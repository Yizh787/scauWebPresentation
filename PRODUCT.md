# Product

## Register

product

## Users

**Students** (primary): University students at Sichuan Agricultural University. They access the system on personal devices (laptops, phones) to take timed exams, review scores, and study from their wrong-answer book. Context: focused, sometimes anxious, need clarity and zero friction during exams.

**Faculty admins**: Professors or teaching staff who manage the question bank, create exams, review student performance, and use AI to generate questions and explain wrong answers. Context: task-oriented, need efficiency and clear data presentation.

## Product Purpose

An online exam system for Sichuan Agricultural University. Students take timed multiple-choice exams and receive instant scoring with AI-powered explanations for wrong answers. Admins manage question banks, create exams (manually or via AI), and track student performance. The system replaces paper-based exam workflows with a reliable, always-available digital alternative.

## Brand Personality

**Modern academic with deliberate delight.** Professional, trustworthy, focused — but not sterile. Key interactions (login, exam submission, loading states, score reveal) get micro-interactions that earn a "wow" without breaking the scholarly calm. The system looks like it was built by someone who cares, not assembled from a template.

Three-word personality: **focused, polished, alive.**

## Anti-references

1. **Default Bootstrap style** — gray-white background, blue primary buttons, identical rounded cards stacked endlessly. Looks like a template, not a product.
2. **Over-flashing animations** — exam context demands restraint. No bouncing buttons, no confetti on every click. Save delight for key moments.
3. **Low information density** — oversized whitespace and huge fonts that make a feature-rich system look empty. Show the work.
4. **Inconsistent visual language** — login page one style, exam page another, admin a third. One vocabulary, every screen.
5. **Zero micro-interactions** — buttons with no hover feedback, forms with no validation states, loading with no indication. Feels abandoned.

## Design Principles

1. **Professional restraint.** The exam context demands focus. Delight lives at key touchpoints (login, submission, score reveal), not on every hover. Earn the "wow" moments by keeping everything else calm.
2. **Consistent visual vocabulary.** Same button styles, same form controls, same card treatment, same spacing scale from login through admin dashboard. One language, every screen.
3. **Show the work.** Dense information when it serves the task (exam lists, question banks, score tables). Don't pad with whitespace to look "designed." Content is the design.
4. **Modern but scholarly.** Contemporary typography, clean layout, purposeful color. Not flashy, not dated. Like a well-run library: organized, inviting, gets out of your way.
5. **Functional completeness.** Every interactive element has proper states: hover, focus, active, disabled, loading, error. No dead ends, no unhandled edge cases.

## Accessibility & Inclusion

Visual-first with basic accessibility. Sufficient color contrast for readability, keyboard navigation for primary flows (login, exam submission, form interactions). Reduced motion respect for users who prefer it. No full WCAG certification required, but don't ship anything that's actively hostile to accessibility.
